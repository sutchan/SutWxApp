 苏铁微信小程序 (SutWxApp) 项目规范

## 目录结构

本目录包含苏铁微信小程序项目的所有规范文档，采用OpenSpec规范系统进行组织和管理。

```
openspec/
├── 项目概述.md          # 项目整体概述和基本信息
├── README.md            # 规范文档索引和目录
├── archive/             # 存档文档
└── specs/               # 核心规范文档
    ├── api/             # API设计规范
    ├── architecture/    # 架构设计规范
    ├── development/     # 开发规范
    ├── features/        # 功能规范
    ├── ops/             # 运维规范
    ├── project/         # 项目规范
    └── testing/         # 测试规范
```

## 核心规范文档

### 1. 项目概述
- **文件**: `项目概述.md`
- **描述**: 项目的整体概述，包括项目背景、目标、技术栈、项目约定等基本信息
- **主要内容**: 项目目的、技术栈、项目约定、业务背景、重要约束、外部依赖

### 2. 架构规范
- **文件**: `specs/architecture/spec.md`
- **描述**: 系统的技术架构设计，包括前端技术栈、后端服务、开发流程、安全设计、性能优化和监控策略
- **主要内容**: 技术栈、架构设计原则、系统分层结构、核心模块架构、数据关系设计、安全设计、性能优化、部署设计、监控与运维

### 3. API规范
- **文件**: `specs/api/spec.md`
- **描述**: API设计和使用规范，包括API结构、设计原则、认证机制、核心接口、错误处理和版本管理
- **主要内容**: 设计原则、API结构、HTTP方法、状态码、认证与授权、请求格式、响应格式、核心API接口、错误处理、版本管理、API测试、安全措施

### 4. 功能规范
- **文件**: `specs/features/spec.md`
- **描述**: 功能模块结构、核心功能和使用方法
- **主要内容**: 设计原则、功能模块结构、核心功能模块（首页、分类、产品、购物车、订单、用户、积分、支付、物流、客服、 admin）、模块间关系、功能模块设计标准

### 5. 测试规范
- **文件**: `specs/testing/spec.md`
- **描述**: 测试策略和测试用例，包括功能测试、性能测试、兼容性测试、回归测试和安全测试
- **主要内容**: 设计原则、测试环境、测试类型、测试用例、测试执行、测试结果统计、测试总结

### 6. 开发规范
- **文件**: `specs/development/spec.md`
- **描述**: 开发环境设置、配置和最佳实践，包括前端和后端环境要求、安装步骤、项目配置和开发指南
- **主要内容**: 环境要求、项目安装、项目配置、开发工具配置、运行项目、代码规范、Git工作流、开发流程、最佳实践、调试技巧、常见问题、代码审查、CI/CD、版本管理、文档管理、协作规范

### 7. 运维与部署规范
- **文件**: `specs/ops/spec.md`
- **描述**: 运维与部署指南，包括环境要求、系统配置、权限管理、部署流程、日常运维、监控方法和故障处理程序
- **主要内容**: 环境要求、系统配置、权限管理、系统架构、部署流程、日常运维、监控与告警、日志管理、数据备份、安全管理、故障处理、系统升级、灾难恢复

### 8. 项目规范
- **文件**: `specs/project/spec.md`
- **描述**: 项目的基本信息、架构设计、开发流程和文档规范，确保项目的一致性、可维护性和可扩展性
- **主要内容**: 项目概述、技术栈、项目结构、开发流程、代码规范、Git工作流、文档规范、质量保证、部署策略、监控与运维、安全策略、性能优化、团队协作、版本管理、变更管理

### 9. 用户指南
- **文件**: `specs/user-guide/spec.md`
- **描述**: 向用户提供如何访问、浏览和使用应用核心功能的详细指导
- **主要内容**: 应用介绍、快速入门、功能使用说明、常见问题、操作指南、最佳实践、注意事项、版本更新、技术支持

## 规范管理

### 文档更新流程
1. 提出文档更新需求
2. 设计文档变更方案
3. 编写文档内容
4. 提交文档审核
5. 文档发布
6. 通知相关人员

### 文档版本管理
- 采用语义化版本号: MAJOR.MINOR.PATCH
- 每次文档更新时，更新版本号和更新日志
- 保留历史版本，便于追溯

### 文档命名规范
- 文档名称使用中文，清晰描述文档内容
- 目录结构采用层级组织，便于查找
- 文件名使用中文字符，避免特殊字符
