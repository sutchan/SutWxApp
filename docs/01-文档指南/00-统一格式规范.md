# 统一格式规范

## 1. 文档结构规范

### 1.1 标题层级
- **一级标题**：使用 `#` 表示，用于文档的主标题
- **二级标题**：使用 `##` 表示，用于文档的主要章节
- **三级标题**：使用 `###` 表示，用于章节下的子章节
- **四级标题**：使用 `####` 表示，用于更细粒度的内容划分

### 1.2 目录
- 每个文档必须包含自动生成的目录
- 目录应位于文档标题下方，使用 `[TOC]` 标记或手动编写

### 1.3 文档元信息
- 每个文档必须包含元信息，包括：
  - 文件名
  - 版本号
  - 更新日期
  - 作者
  - 描述
- 元信息应放在文档顶部，使用 HTML 注释格式

## 2. 内容格式规范

### 2.1 文本格式
- 使用中文标点符号
- 段落之间空一行
- 避免过长的行，建议每行不超过 100 个字符

### 2.2 列表
- **无序列表**：使用 `-` 或 `*` 表示
- **有序列表**：使用数字加 `.` 表示
- 列表项之间空一行
- 列表项内部可以嵌套其他列表

### 2.3 代码块
- 使用三个反引号 `` ``` `` 包裹代码块
- 代码块必须指定语言类型，如 `` ```javascript ``
- 行内代码使用单个反引号 ` 包裹

### 2.4 表格
- 使用 Markdown 表格语法
- 表格标题使用加粗字体
- 表格列对齐方式统一使用左对齐

### 2.5 链接和图片
- **链接**：使用 `[链接文本](链接地址)` 格式
- **图片**：使用 `![图片描述](图片地址)` 格式
- 外部链接必须包含完整的 URL
- 内部链接使用相对路径

### 2.6 强调和引用
- **强调**：使用 `**加粗**` 或 `*斜体*` 表示
- **引用**：使用 `>` 表示
- **警告/注意**：使用 `> [!WARNING]` 或 `> [!NOTE]` 格式

## 3. 命名规范

### 3.1 文件命名
- 使用小写字母和连字符 `-` 分隔单词
- 文件名应简洁明了，反映文档内容
- 示例：`01-项目简介.md`

### 3.2 目录命名
- 使用数字前缀 + 目录名称的格式
- 数字前缀用于排序，如 `00-项目概览`
- 目录名称应简洁明了，反映目录内容

## 4. 编码和格式

### 4.1 文件编码
- 所有文件必须使用 UTF-8 无 BOM 格式

### 4.2 换行符
- 所有文件必须使用 Unix LF 格式

### 4.3 缩进
- 使用 2 个空格进行缩进
- 避免使用制表符

## 5. 文档维护规范

### 5.1 更新频率
- 文档应与代码同步更新
- 每次代码变更涉及文档内容时，必须更新相应文档

### 5.2 版本管理
- 文档版本号使用语义化版本规范（MAJOR.MINOR.PATCH）
- 每次文档更新必须更新版本号和更新日期

### 5.3 审核机制
- 文档更新必须经过审核
- 审核通过后才能合并到主分支
