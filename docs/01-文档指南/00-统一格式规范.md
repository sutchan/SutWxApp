﻿﻿﻿# 统一格式规范\n\n## 1. 文档结构规范\n\n### 1.1 标题层级\n- **一级标题**：使用 `#` 表示，用于文档的主标题\n- **二级标题**：使用 `##` 表示，用于文档的主要章节\n- **三级标题**：使用 `###` 表示，用于章节下的子章节\n- **四级标题**：使用 `####` 表示，用于更细粒度的内容划分\n\n### 1.2 目录\n- 每个文档必须包含自动生成的目录\n- 目录应位于文档标题下方，使用 `[TOC]` 标记或手动编写\n\n### 1.3 文档元信息\n- 每个文档必须包含元信息，包括：\n  - 文件名\n  - 版本号\n  - 更新日期\n  - 作者\n  - 描述\n- 元信息应放在文档顶部，使用 HTML 注释格式\n\n## 2. 内容格式规范\n\n### 2.1 文本格式\n- 使用中文标点符号\n- 段落之间空一行\n- 避免过长的行，建议每行不超过 100 个字符\n\n### 2.2 列表\n- **无序列表**：使用 `-` 或 `*` 表示\n- **有序列表**：使用数字加 `.` 表示\n- 列表项之间空一行\n- 列表项内部可以嵌套其他列表\n\n### 2.3 代码块\n- 使用三个反引号 `` ``` `` 包裹代码块\n- 代码块必须指定语言类型，如 `` ```javascript ``\n- 行内代码使用单个反引号 ` 包裹\n\n### 2.4 表格\n- 使用 Markdown 表格语法\n- 表格标题使用加粗字体\n- 表格列对齐方式统一使用左对齐\n\n### 2.5 链接和图片\n- **链接**：使用 `[链接文本](链接地址)` 格式\n- **图片**：使用 `![图片描述](图片地址)` 格式\n- 外部链接必须包含完整的 URL\n- 内部链接使用相对路径\n\n### 2.6 强调和引用\n- **强调**：使用 `**加粗**` 或 `*斜体*` 表示\n- **引用**：使用 `>` 表示\n- **警告/注意**：使用 `> [!WARNING]` 或 `> [!NOTE]` 格式\n\n## 3. 命名规范\n\n### 3.1 文件命名\n- 使用小写字母和连字符 `-` 分隔单词\n- 文件名应简洁明了，反映文档内容\n- 示例：`01-项目简介.md`\n\n### 3.2 目录命名\n- 使用数字前缀 + 目录名称的格式\n- 数字前缀用于排序，如 `00-项目概览`\n- 目录名称应简洁明了，反映目录内容\n\n## 4. 编码和格式\n\n### 4.1 文件编码\n- 所有文件必须使用 UTF-8 无 BOM 格式\n\n### 4.2 换行符\n- 所有文件必须使用 Unix LF 格式\n\n### 4.3 缩进\n- 使用 2 个空格进行缩进\n- 避免使用制表符\n\n## 5. 文档维护规范\n\n### 5.1 更新频率\n- 文档应与代码同步更新\n- 每次代码变更涉及文档内容时，必须更新相应文档\n\n### 5.2 版本管理\n- 文档版本号使用语义化版本规范（MAJOR.MINOR.PATCH）\n- 每次文档更新必须更新版本号和更新日期\n\n### 5.3 审核机制\n- 文档更新必须经过审核\n- 审核通过后才能合并到主分支\n\n## 6. 示例\n\n```markdown\n<!--\n文件名: example.md\n版本号: 1.0.0\n更新日期: 2025-11-30\n作者: Sut\n描述: 示例文档，展示统一格式规范\n-->\n\n# 示例文档\n\n## 1. 概述\n\n这是一个示例文档，用于展示统一格式规范的使用方法。\n\n## 2. 主要内容\n\n### 2.1 列表示例\n\n- 无序列表项 1\n- 无序列表项 2\n  -
