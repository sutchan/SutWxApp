# 开发计划

## 1. 目标与范围
- 完成电商闭环与积分基础能力，提升稳定性与用户体验。

## 2. 版本规划（SemVer）
- v1.5.x：问题修复与性能优化、积分抵扣完善、消息稳定性提升。
- v1.6.0：会员权益、任务中心完善、活动矩阵（满减/券/秒杀）。
- v1.7.0：多店铺与运营后台强化、风控策略升级。

## 3. 里程碑
- M1（本月）：
  - 完成积分抵扣/回退策略与接口联调。
  - 优化下单与支付链路的并发与幂等处理。
  - 增加订阅消息失败重试与告警。
- M2（下月）：
  - 会员等级与权益联动积分体系。
  - 任务中心（每日/每周/每月）上线与运营配置。
  - 活动模块基础能力与券系统接入。

## 4. 风险与应对
- 并发一致性：加锁与幂等键；失败可重试与审计记录。
- 支付稳定性：通道降级与对账补偿；超时与回退策略。
- 风控：限频与设备指纹；黑白名单与灰度发布。

## 5. 度量与验收
- 接口错误率 < 0.5%；订单成功率 ≥ 99%。
- 积分扣减/回退一致性 100%，异常有流水与告警。
- 消息送达率 ≥ 98%，失败自动重试与人工兜底。
