## 3. 使用说明

1. 开发者应首先阅读《01-开发环境配置.md》，完成开发环境搭建
2. 然后阅读《02-架构设计文档.md》，了解系统架构
3. 根据需要阅读相关API文档和开发指南
4. 开发过程中应遵循《08-代码规范.md》

## 4. 开发流程

1. 需求分析：理解需求，确定技术方案
2. 设计：进行架构设计和详细设计
3. 开发：按照设计文档进行开发
4. 测试：进行单元测试和集成测试
5. 代码审查：提交代码审查，修复问题
6. 部署：部署到测试环境，进行验收测试
7. 发布：发布到生产环境
