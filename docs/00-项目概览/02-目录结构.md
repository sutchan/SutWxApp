## 1. 项目目录结构

SutWxApp项目采用清晰的目录结构，便于代码管理和维护。以下是项目的主要目录结构：

```
SutWxApp/
├── app.js              # 小程序入口文件
├── app.json            # 全局配置
├── app.wxss            # 全局样式
├── assets/             # 静态资源
│   └── images/         # 图片资源
├── components/         # 自定义组件
├── images/             # 图片文件
├── pages/              # 页面文件
│   ├── article/        # 文章相关页面
│   ├── user/           # 用户相关页面
│   └── ...             # 其他功能模块页面
├── services/           # 服务层
├── tests/              # 测试文件
├── utils/              # 工具类
└── docs/               # 项目文档
    ├── 00-项目概览/       # 项目整体介绍
    ├── 01-文档指南/       # 文档编写规范和模板
    ├── 02-管理员指南/     # 管理员相关文档
    ├── 03-开发者指南/     # 开发者相关文档
    ├── 04-用户指南/       # 用户使用指南
    ├── 05-功能模块/       # 功能模块说明
    ├── 06-常见问题/       # FAQ和故障排除
    ├── 07-测试文档/       # 测试相关文档
    ├── 08-TODO计划/       # 开发计划和TODO清单
    ├── 09-运维与部署/     # 运维和部署指南
    ├── 10-归档资料/       # 归档文档
    └── 11-优化建议/       # 优化建议和未来展望
```

## 2. 主要目录说明

### 2.1 根目录

- **app.js**：小程序的入口文件，包含小程序的生命周期函数
- **app.json**：全局配置文件，用于配置小程序的页面路径、窗口样式、导航栏等
- **app.wxss**：全局样式文件，定义小程序的全局样式

### 2.2 assets/

存放项目的静态资源，如图片、音频、视频等。

- **images/**：存放图片资源

### 2.3 components/

存放自定义组件，用于实现组件化开发，提高代码复用性。

### 2.4 images/

存放小程序中使用的图片文件。

### 2.5 pages/

存放小程序的页面文件，每个页面通常包含四个文件：.js、.json、.wxml、.wxss。

- **article/**：文章相关页面
- **user/**：用户相关页面
- 其他功能模块页面

### 2.6 services/

存放服务层代码，用于封装API请求、数据处理等逻辑。

### 2.7 tests/

存放测试文件，用于测试代码的正确性和稳定性。

### 2.8 utils/

存放工具类代码，如日期处理、字符串处理、网络请求等。

### 2.9 docs/

存放项目文档，包含项目概览、文档指南、管理员指南、开发者指南等。

## 4. 目录使用规范

### 4.1 文件命名规范

- 页面文件：使用小驼峰命名法，如 `userInfo.js`
- 组件文件：使用小驼峰命名法，如 `lazyImage.js`
- 服务文件：使用小驼峰命名法，如 `userService.js`
- 工具文件：使用小驼峰命名法，如 `dateUtil.js`

### 4.2 目录创建规范

- 新功能模块应创建对应的目录
- 目录名称应简洁明了，反映模块功能
- 遵循现有的目录结构和命名规范
